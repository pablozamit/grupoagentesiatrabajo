## Sistema de Tags

Los hashtags reflejan **para qué sirve** desde el punto de vista del alumno. Cada tag tiene su propia nota con las entradas relacionadas.

### Por tipo de contenido

| Tag | Descripción |
|-----|-------------|
| #contenido | Teoría y explicación |
| #vibecoding | Metodología de desarrollo |
| #modelos | Modelos de IA |
| #basededatos | Bases de datos |
| #proyecto | Ejercicio práctico |

### Por dominio (qué puede crear/hacer)

| Tag | Descripción |
|-----|-------------|
| #juegos | Videojuegos |
| #3d | Gráficos 3D |
| #diseno | Diseño UI/UX |
| #video | Video y edición |

### Por funcionalidad

| Tag | Descripción |
|-----|-------------|
| #agente | Agentes de IA |
| #hosting | Publicar en internet |
| #ciencia | Ciencia e investigación |
| #investigacion | Investigación académica |

---

## Reglas para tags

1. **Los hashtags van al FINAL del documento**
2. **Mínimo 2-3 tags por nota** (diferentes categorías)
3. **Debe ser entendible por el aluno**: #juegos sí, #api no
4. **No usar términos técnicos** que el alumno no entiende
